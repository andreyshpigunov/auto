    Installing

    Github

    Download and connect files manually: https://github.com/andreyshpigunov/auto

    It is better to implement styles in your project using scss, then all the settings will be available. if you connect a css file, you will have to manually override the settings with styles.

    CDN

    Current versions of files are available at the following links:

    https://cdn.jsdelivr.net/gh/andreyshpigunov/auto@latest/css/auto.css
    https://cdn.jsdelivr.net/gh/andreyshpigunov/auto@latest/js/auto.js



    Auto.css

    Helpers

    Extend the functionality of elements.

    Links

    .no-visited — the class removes the selection from the link with this data :visited
    .pseudo — pseudo-link, underlined with dots
    .no-border — link without underscores

    If the link contains an image, the underscore is removed automatically (class .no-border added to link), you can return it by adding a class .border.

    Helpers classes

    .unselectable — disables the selection of an element and all its descendants
    .clear — clearfix
    .uppercase — uppercase text
    .lowercase — lowercase text
    .capitalize — the first characters of words are uppercase
    .nowrap — prohibiting line wrapping
    .hyphens — automatic hyphenation (works only in modern browsers)
    .video — wrapper for iframe, object и embed, proportionally scale filter
    .break-word — transfers non-portable text constructs
    .visuallyhidden — hides the block in a special way
    .hidden — assigns to a block display: none
    .hidden-next — assigns to the next block display: none
    .invisible — assigns to a block visibility: hidden
    .table-responsive — horizontal scrolling of the table if it doesn't fit
    .up — raises the element above the context (adds to it z-index: 2)

    Actions classes

    .sync-hover — adding a class synchronously .hover to links with the same href
    .is-appeared — when it appears in the viewport, the element is assigned classes .appeared and .visible. When disappearing from viewport class .visible disappear, .appeared remains.

    Grid

    Adaptive flexbox blocks that automatically adjust to the line width. The whole principle of operation is based on flexboxes, you can very flexibly manipulate the grid in CSS, if there are not enough standard features.

    In the auto.scss you can see the custom parameters:
    $grid-columns: 12 — number of columns in the grid
    $grid-gutters-large: 1.2vw — margins inside blocks on different screen sizes
    $grid-gutters-medium: 1.2vw
    $grid-gutters-small: 1.2vw

    [size] — sizes: small, medium or large.

    .modules — modules group
    .modules.-nearby — remove paddings between columns
    .modules.-line — remove padding under row

    .module — one module
    .module.[size]-[0..12] — module width in columns
    .module.[size]-auto — auto module width
    .module.[size]-offset-[0..12] — module offset
    .module.[size]-order-[0..12] — module order
    .module.[size]-order-first — module order first
    .module.[size]-order-last — module order last
    .modules-linebreak — modules linebreak
    .modules-colbreak — modules colbreak

    Adaptivity

    Screen sizes

    Size Screen width, px
    small 0..500
    medium 501..1000
    large 1001..∞

    CSS

    Hide elements on differente screen sizes.

    .large-hide — hide on large screen
    .medium-hide — hide on medium screen
    .small-hide — hide on small screen

    SASS

    Using in SCSS

    @media #{$screen-large} {...}
    @media #{$screen-medium} {...}
    @media #{$screen-small} {...}
    @media #{$screen-medium}, #{$screen-small} {...} — mixing

    Cleaner variant:

    @include screen(large) {...}
    @include screen(medium) {...}
    @include screen(small) {...}
    @include screen(medium, small) {...}

    Кнопки

    Чтобы создать кнопку, надо добавить элементу класс .button.

    .button .button.-bold .button.-bordered
    .button.-large .button.-small .button.-xsmall

    Адаптивность

    Функция срабатывает только при первой загрузке страницы.

    if(auto.adaptive.large){ ... }
    if(auto.adaptive.medium){ ... }
    if(auto.adaptive.small){ ... }

    Примеры

    Анимации

    Запускает функцию с анимацией или управляет классом элемента при достижении им или его элементом-триггером определенной точки прокрутки.

    Инициализация

    TriggerID — указатель на ID эелемента, у которого мы отслеживаем положение
    Start — точка начала анимации (высота от верхнего края экрана в px или vh)
    End — точка окончания анимации (высота от верхнего края экрана в px или vh)
    FunctionName — название функции, выполнение которой начинается, когда элемент движется между Start и End
    Class — класс, присваиваемый ссылке, когда Target находится между Start и End
    ClassRemove — удалять ли Class у елемента после выхода его из зоны анимации, по умолчанию false

    Если элемент сам является триггером, TriggerID указывать не нужно.

    Функция FunctionName получает параметр со всеми полями элемента, описанными выше и параметром Progress, который изменяется от 0 до 1, в процессе прокрутки страницы и перемещения элемента.

    Пример

    Скроллер

    Прокручивает страницу до элемента с указанным ID.
    Автоматически применяется к элементам с data-scroll.

    Инициализация

    Target — ID цели
    Offset — смещение в пикселях (по умолчанию 0)
    Speed — скорость прокрутки (по умолчанию 500 мс)
    ClassActive — класс, присваиваемый ссылке, когда Target находится в зоне viewport

    API

    auto.scrollTo(target, offset, speed)

    target — ID цели
    offset — скорость прокрутки (по умолчанию 500 мс, необязательные параметр)
    speed — смещение в пикселях (по умолчанию 0, необязательные параметр)

    Загрузчик

    Запускает функцию подгрузки данных при прокрутке до конца блока.
    Автоматически применяется к элементам с data-loadmore.

    Инициализация

    FunctionName — название функции, выполняющей загрузку
    Offset — смещение в пикселях от нижнего края страница (по умолчанию 0)

    В функцию при каждом ее запуске передается инкрементный параметр page.

    Ленивая загрузка

    Загрузка изображений при их появлении в viewport.
    Работает с помощью метода IntersectionObserver, только в современных браузерах.
    Если IntersectionObserver не поддерживается, картинки загружаются обычным способом.
